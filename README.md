# 无人机边缘计算调度与监控系统 (UAV Edge Computing System)

一个全方位的“赛博朋克风”仪表盘系统，模拟并展示了在地面数据源 (用户/IoT)、高空无人机边缘信关站 (Tier-2 集群) 以及远程云核心兜底服务器 (Tier-3) 之间的智能分布式算力卸载。

## 🚀 核心特性
- **实时数据交互雷达 (Active Topology)**：在带有物理拓扑网格的雷达中，实时俯瞰从指挥中心派遣到各无人机计算节点的高性能任务链路。
- **动态算力网格管理 (Dynamic Cluster Management)**：支持在集群面板中热拔插扩展边缘网络。动态部署新的无人机节点，并可模拟调整其算力规模 (CPU/内存/网络带宽)。
- **全生命周期任务流水线 (Asynchronous Task Workloads)**：模拟图像处理、AI推理和遥测数据包流，并精准追踪任务端到端的瀑布流耗时 (排队/调度/计算/传输/损耗)。
- **智能工作窃取与卸载算法 (Algorithmic Load Balancing)**：系统内置物理拓扑最短路径 (Geo)、能耗优先 (Energy) 和最低延迟 (Latency) 多种调度算法，并支持边缘过载时的无人机“工作窃取 (Work-Stealing)”互助协议。
- **故障演练与云端兜底 (Failover / Tiering)**：内置物理电池功耗与坠毁 (DOWN) 模拟。当边缘层算力被全部击穿或坠毁时，系统能优雅且平滑地将海量超载流量通过广域网引流至无限算力的 `CLOUD-SERVER`。

## 🛠 环境依赖

运行该项目的前后端微服务架构，请确保您已开启以下环境：
- **Node.js** (推荐 v16.0 及以上版本)
- **Java 17** (或更高版本，核心调度依赖)
- **MySQL 8.0**
- **Redis Server** (必须启动，用于持久化的优先级高性能异步任务队列)

## 📦 本地启动指南

### 1) 启动后端调度微服务 (Spring Boot)
1. 进入 `backend` 目录：
   ```bash
   cd backend
   ```
2. 确保您的 **MySQL** 和 **Redis** 已经在本地以默认端口 (`3306` 和 `6379`) 运行。
3. 在 `src/main/resources/application.properties` 中核对并设置您的数据库账号密码。
4. 使用 Maven Wrapper 原生启动：
   ```bash
   ./mvnw clean spring-boot:run
   ```
   > 服务器将在 `http://localhost:8080` 起飞。前端 API 路由规范定义于 `/api/` 路径下。

### 2) 启动前端态势感知面板 (Vue 3 + Vite)
1. 另开一个终端，进入 `frontend` 工程目录：
   ```bash
   cd frontend
   ```
2. 安装所有依赖包：
   ```bash
   npm install
   ```
3. 唤醒本地的 Vite 高速热模加载服务器：
   ```bash
   npm run dev
   ```
4. 打开浏览器，访问这个视觉极其硬核的控制台：**`http://localhost:5173`**。

## 🎮 平台作战指南详解

1. **防御中心 (OP-CENTER) 连线**
   - 进入主页，键入您的指挥官代号并完成拓扑基站在网格中的建联坐标定标，随后系统会自动握手 WebSocket 通讯。
2. **态势感知总览 (Dashboard Overview)**
   - 雷达界面将瞬间点亮您的指挥中心 (👤) 以及在空域内游荡巡航的边缘无人机阵列 (✈️)。
   - 使用滚轮随意放大/缩小网格刻度，长按鼠标自由平移这片电子战区的沙盘。无人机图标均可通过拖拽锚定位置。
3. **倾泻计算任务 (Dispatching Tasks)**
   - 在左上角点击蓝色的 **"🚀 发布突发负载" (Issue Task)**。
   - 随心所欲地拉高并发峰值和数据载荷权重。您将看到几MB的轻型任务被敏捷的无人机原生吞下；而上百MB的深水炸弹则会因为算力溢出，被无情地分配给位于远端 (0, 0) 地平线之外的 Tier-3 云端集群。
4. **集群算力扩增 (Network Expansion)**
   - 切入左侧菜单的 **"边缘网格信标 (Cluster)"** 频道。
   - 当任务遭遇堵塞，果断点击 **"+ 部署新站"**。系统会以毫秒级的速度热加载一台崭新的无人机入网，其 CPU 核心和内存将立刻向调度系统的红黑树中注册，加入排队抢单。
5. **审计微观时序 (Auditing Lifecycle)**
   - 前往 **"战役事件追溯 (Trace)"** 界面调出极为细致的法证级流水。每一毫秒的排队、每一焦耳的耗电、网络波动造成的延迟，均会被切割成可视化时间轴。这里是寻找真实世界性能瓶颈的绝佳手术台！
